주제는 무엇인가요?

  • python으로 네트워크 장비 테스트 자동화 하기

왜 이 발표를 계획 하게 되었나요?

  • 기존 사내에서 수동으로 테스트를 진행하던 테스트 엔지니어였습니다. 테스트 자동화에 대한 인사이트를 얻고자, 지난 PyCon 2017 각 부스와 Open Space를 돌아다녔습니다. 하지만 Web service 혹은 개발 자체에 대한 이야기들이 주를 이루었고, 테스트 혹은 테스트 프레임워크 개발, 네트워크에 대한 이야기는 전무해 아쉬웠습니다.

  • Python은 네트워크 장비 운영 및 설정, 테스트에서도 활발히 움직임을 보이고 있고, 무엇보다 쉽게 접근 할 수 있는 언어기 때문에, 누군가 저처럼 테스트 자동화 개발 자체에 고민을 가지고 왔지만 약간 아쉬움을 느끼신 분들이 있을 것이라 생각했습니다. 그래서 제가 스스로 “우리 같은 사람도 있어요!”를 알리고 싶은 마음으로 참여하게 되었습니다.

  • 특히나 이번 PyCon의 주제도 ‘다양성’ 이기에 해당 주제도 좋은 의미가 있을 것이라 생각했습니다.

대상 청중은 누구인가요?

  • 자동화를 도입하고 계신 CLI 기반 소프트웨어 테스트 엔지니어 분들

  • 아직 Python이 무서운 수동 테스트 엔지니어 분들

  • Python을 통한 네트워크 장비 테스트 방법이 궁금하신 분들

발표를 통해서 청중이 어떤 것을 얻을 수 있나요?

  • 개발 경험이 전혀 없던 제가 1년동안 네트워크 자동화 테스트 프레임워크 개발 과정을 통해 얻은 경험을 공유하고자 합니다.

  • 청중은 Python을 통한 네트워크 장비 테스트 방법, 넓게는 CLI 기반의 소프트웨어 테스트 접근 방식에 대한 인사이트를 얻을 수 있을 것으로 예상합니다.
